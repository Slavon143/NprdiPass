# Backup & Restore

## Overview

NordiPass uses custom Artisan commands for backup, restore, verification, and pruning. No external backup package is required.

| Component | Included | Notes |

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `BACKUP_ENABLED` | `true` | Enable backup command |
| `BACKUP_DISK` | `local` | Filesystem disk for backup storage |
| `BACKUP_PATH` | `nordipass/backups` | Path on the disk |
| `BACKUP_DATABASE_ENABLED` | `true` | Include database in backup |
| `BACKUP_FILES_ENABLED` | `true` | Include files in backup |
| `BACKUP_DATABASE_BINARY` | (auto-detect) | Path to `mysqldump` binary |
| `BACKUP_COMPRESSION` | `gzip` | Compression algorithm |
| `BACKUP_VERIFY_AFTER_CREATE` | `true` | Run checksum verification |
| `BACKUP_RETENTION_DAILY` | `7` | Daily backups to keep |
| `BACKUP_RETENTION_WEEKLY` | `4` | Weekly backups to keep |
| `BACKUP_RETENTION_MONTHLY` | `3` | Monthly backups to keep |
| `BACKUP_LOCK_MINUTES` | `180` | Lock duration for backup |
| `BACKUP_MAX_AGE_HOURS` | `24` | Max age before notification |
| `BACKUP_ENCRYPTION_ENABLED` | `false` | Encryption status |

## Commands

```bash
# Full backup
php artisan nordipass:backup

# Database only
php artisan nordipass:backup --database-only

# Files only
php artisan nordipass:backup --files-only

# Dry run (validate config only)
php artisan nordipass:backup --dry-run

# Skip post-backup verification
php artisan nordipass:backup --no-verify
```

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | General failure |
| 2 | Invalid configuration (backup disabled) |
| 3 | Already running (lock acquired) |
| 4 | Verification failed |

## Backup storage

### Local

```env
BACKUP_DISK=local
```

Backups are stored in `storage/app/private/nordipass/backups/<backup-id>/`.

### Production offsite

For production, configure a remote filesystem disk:

```env
BACKUP_DISK=s3   # or sftp
```

A backup stored on the same server is **not** a disaster recovery solution.

## Artifacts

Each backup consists of:

```
<backup-id>/
  database.sql.gz        # MySQL dump, gzipped
  files.tar.gz           # Application files archive
  manifest.json          # Versioned metadata
```

## Retention

| Policy | Count | Rule |
|--------|-------|------|
| Daily | 7 | Last verified backup per day |
| Weekly | 4 | Monday backups |
| Monthly | 3 | First day of month backups |

The last verified backup is always preserved.

```bash
# Dry run
php artisan nordipass:backup-prune --dry-run

# Prune
php artisan nordipass:backup-prune
```

## Verification

```bash
php artisan nordipass:backup-verify <backup-id>
```

Verification checks:
- Manifest exists and is valid JSON
- Manifest version is supported
- Status is `complete`
- All artifact checksums match
- All artifact files exist

## Restore

```bash
# Restore to current database
php artisan nordipass:restore <backup-id> --force

# Restore to a different connection
php artisan nordipass:restore <backup-id> --target-connection=mysql_testing --force
```

### Safety

- Production requires `--force` and `--confirm-production-restore`
- Typed confirmation (`RESTORE NORDIPASS PRODUCTION`) required in production
- Checksums verified before restore
- Pre-restore safety backup recommended
- Maintenance mode recommended

### Order

1. Select verified backup
2. Verify checksums
3. Create pre-restore safety backup
4. Enable maintenance mode
5. Stop queue workers
6. Restore database
7. Restore files
8. Clear caches
9. Run `php artisan nordipass:restore-verify`
10. Restart workers
11. Disable maintenance mode
12. Verify `/up` and `/ready`

## Restore verification

```bash
php artisan nordipass:restore-verify
```

Checks:
- Database connection
- Migrations table
- Critical tables exist (companies, users, company_user, company_invitations)
- Tables have records
- Foreign key integrity (no orphaned references)
