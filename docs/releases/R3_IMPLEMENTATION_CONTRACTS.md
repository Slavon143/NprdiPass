## Shared Contract Rules

- Web and API controllers must call the same application/domain Actions.
- Published Passport APIs and public pages must read immutable published versions, never mutable draft data.
- Permissions, entitlements, and feature flags are separate concepts and must not be substituted for each other.
- Jobs must carry tenant context, be idempotent, and be safe after related entity deletion.
- Migrations use expand → backfill → dual-read/write where required → switch → contract in a later release.
